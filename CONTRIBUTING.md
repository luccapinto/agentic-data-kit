# Contributing to the Agentic Data Kit

> This kit works out of the box. If you want to extend it, here's how — and, just as
> importantly, when *not* to.

---

## The Golden Rule

**Only edit `.agent/`.** Everything under `.github/`, `.claude/`, `.opencode/`, and the root
`AGENTS.md` is auto-generated by `scripts/sync_agents.py` and will be overwritten.

```bash
# After editing anything in .agent/, regenerate every platform:
python scripts/sync_agents.py
```

---

## Before you add an agent: should it exist?

Less is more. Extra agents and verbose instructions cost the model attention and make routing
ambiguous — both degrade results. Apply these gates before creating anything:

- **Skill suffices?** If the capability is narrow knowledge (a format, a checklist, a tool),
  make it a **skill**, not an agent. Skills load only when relevant (progressive disclosure),
  so they're nearly free until used.
- **Already covered?** If an existing agent's domain overlaps, extend that agent instead.
- **LLM-native?** Don't write down what a capable model already does well ("write clean code",
  "use CTEs"). Capture only *domain- or org-specific* context.

If a request fails these gates, the right answer is to **not** add the artifact.

---

## How to add something

1. **Create** the file under the right `.agent/` subfolder:
   - Agent → `.agent/agents/<name>.md`
   - Skill → `.agent/skills/<name>/SKILL.md` (+ optional `scripts/`, reference files)
   - Workflow → `.agent/workflows/<name>.md`
   - Global rule → `.agent/rules/rules.md`
2. **Sync** → `python scripts/sync_agents.py`
3. **Use** — the agent/skill/command appears in each tool automatically.

---

## Conventions

**Agents** (`.agent/agents/<name>.md`) — keep them lean and high-signal:

```yaml
---
name: my-agent              # lowercase-hyphenated
description: One sharp sentence on what it does + trigger keywords (used for routing).
tools: Read, Grep, Glob, Bash, Edit, Write
model: inherit
skills: skill-a, skill-b    # optional; loaded on demand
---
```
Body = identity (1–2 lines) → domain rules → handoffs → out-of-scope. Cut anything generic.

**Skills** — `SKILL.md` with `name` + `description` frontmatter. Keep the body under ~500 lines;
move long reference material to separate files in the skill folder and link to them.

**Workflows** — `description` frontmatter; concise, numbered steps.

**Language** — write all instructions in **English** for portability; the assistant replies in
the user's language at runtime.

## Folder structure

```
.agent/            ← SOURCE OF TRUTH (edit here)
├── agents/        ← specialist personas
├── skills/        ← deep, on-demand knowledge modules
├── workflows/     ← slash-command procedures
├── rules/         ← global rules (rules.md)
└── scripts/       ← optional validation helpers

.github/  .claude/  .opencode/  AGENTS.md   ← AUTO-GENERATED (do not edit)
```
