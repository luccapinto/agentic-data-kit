# Data-story outline

Fill this in **before** building slides. Lead with the conclusion; each beat is one slide.
Adapt the beats to the genre (exploratory analysis · dashboard readout · business plan · proposal).

## 0. Frame
- **Audience:** (exec / technical / mixed)
- **Decision this enables:** (what should they do after?)
- **Format:** deck / interactive site / PDF — and why
- **One-line takeaway (BLUF):**

## 1. The ask / question
What are we answering, and why now?

## 2. Bottom line (BLUF)
The single most important conclusion — stated as an insight, not a metric.

## 3–N. Evidence (one insight per beat)
For each: **insight title** + the chart/table that proves it + **source · grain · date range · filters**.
- Beat A — insight: …
- Beat B — insight: …
- Beat C — insight: …

## So what
Implications: what changes because of this.

## Recommendation / next steps
Action · owner · timeline · what we'd measure to confirm.

## Appendix
Methodology, assumptions, data lineage, edge cases. Detail the main flow skipped.

---
**Genre tweaks**
- *Exploratory analysis:* emphasize what's surprising; show what you ruled out.
- *Business plan:* market → problem → solution → model → traction → ask.
- *Proposal:* current pain → proposed change → cost/benefit → risks → decision needed.
